# ER-0045.1 — Conduit I/O Operations and Schema Definitions

## Roles

- Implementation Engineer: drafts and implements changes
- System Engineer: reviews, tests, and verifies
- Note: Only the System Engineer may mark an ER as Verified.

## ER Metadata

- ER ID: ER-0045.1
- Title: Conduit I/O Operations and Schema Definitions
- Status: Proposed
- Date: 2026-03-07
- Owners: Mike
- Type: Enhancement

## Context

- Kernel-grade execution needs I/O primitives exposed as typed operations.
- Conduit dispatch currently resolves operations but lacks kernel I/O execution plumbing.

## Goals

- Define schema types and operation signatures for kernel I/O primitives.
- Provide Conduit execution plumbing for I/O operations aligned with CEO wait/wake.
- Ensure errors are explicit and deterministic for invalid arguments or closed channels.

## Non-Goals

- Full device driver or hardware I/O APIs.
- Advanced networking or transport protocol support.

## Scope

- In scope: schema definitions for I/O primitives and operations.
- In scope: Conduit execution mapping to CEO IoReactor and TaskComms.
- Out of scope: persistent handles or driver backends.

## Requirements

- Functional: Conduit operations can open/close channels and datagram ports.
- Functional: Conduit operations can send/recv bytes and await readability.
- Non-functional: operation signatures are validated before execution.

## Proposed Approach

- Add minimal I/O type definitions and operation signatures to the schema bootstrap.
- Introduce a Conduit execution path that maps dispatch matches to CEO I/O primitives.
- Centralize error mapping for invalid task IDs, closed handles, and invalid payloads.

## Acceptance Criteria

- Operation definitions exist for `open_channel`, `open_datagram`, `send`, `recv`, `await_readable`, and `close`.
- Conduit dispatch can execute each I/O operation and return explicit results.
- Unit tests cover schema registration and Conduit execution error paths.

## Risks / Open Questions

- Defining stable I/O handle representations without persistence.
- The minimal operation set required for the v1 kernel demo.

## Dependencies

- ER-0015 Phase5 Comms Primitives.
- ER-0016 Phase5 CEO I/O Reactor.
- ER-0044 Capability Hooks and Policy Plumbing.

## Verification Plan

- Tests: schema registration tests and Conduit execution unit tests.
- Manual: run a Conch command that exercises a Conduit I/O operation and verify output.
