# GitHub Issue Labels (Recommended)

This repo uses labels to keep issues consistent and easy to filter. The list below is small and
purpose-built for IrisOS.

## Type Labels

- `ar` Architecture Recommendation
- `er` Engineering Request
- `dr` Defect Report
- `documentation` Documentation or process
- `how-to` Usage guidance or walkthroughs

## Status Labels

- `status:proposed`
- `status:accepted`
- `status:in-progress`
- `status:done`

## Area Labels (Use as needed)

- `area:conch`
- `area:refract`
- `area:referee`
- `area:viz`
- `area:ceo`
- `area:comms`
- `area:build`

## Guidance

- Every issue should have one **type** label.
- Use **status** labels for AR/ER/DR issues.
- Add **area** labels only when they help triage.
